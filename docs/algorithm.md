# Математическое обоснование алгоритма
 
## 1. Модель спроса
 
Используем **линейную модель спроса** (достаточна для MVP):
 
```
Q(p) = A − B·p
```
 
где:
- `Q` — количество проданных единиц в день
- `p` — цена товара
- `A` — «автономный спрос» (при p→0 сколько бы купили)
- `B` — ценовая чувствительность (насколько падает спрос при росте цены на 1 руб)
**Параметры A и B** оцениваются через OLS-регрессию на исторических данных.
 
---
 
## 2. Функция прибыли
 
```
π(p) = Выручка − Затраты
     = p · Q(p) − cost · Q(p)
     = (p − cost) · Q(p)
     = (p − cost) · (A − B·p)
```
 
---
 
## 3. Нахождение оптимума
 
Берём производную по p и приравниваем к нулю:
 
```
dπ/dp = (A − B·p) + (p − cost)·(−B) = 0
      = A − B·p − B·p + B·cost = 0
      = A − 2B·p + B·cost = 0
```
 
Решаем:
 
```
2B·p = A + B·cost
p* = (A + B·cost) / (2B)
```
 
Это **аналитический оптимум** — точное решение без итераций.
 
---
 
## 4. Ценовые ограничения
 
Оптимальная цена всегда зажимается в диапазон:
 
```
p_min = cost × 1.05    (минимальная маржа 5% — не продаём в убыток)
p_max = base_price × 1.30  (не поднимаем более чем на 30%)
 
p_final = clip(p*, p_min, p_max)
```
 
---
 
## 5. Интерпретация R²
 
R² (коэффициент детерминации) показывает, насколько хорошо линейная модель
описывает исторические данные:
 
| R² | Интерпретация |
|----|---------------|
| > 0.7 | Отличная модель |
| 0.4–0.7 | Приемлемо |
| < 0.4 | Слабая — данных мало или нужна нелинейная модель |
 
---
 
## 6. Почему не максимизировать выручку?
 
Максимум **выручки** (p·Q) даёт другую цену:
```
p_revenue* = A / (2B)
```
 
Это всегда **ниже** цены максимума прибыли, потому что не учитывает себестоимость.
Мы максимизируем **прибыль** — это правильная бизнес-цель.
