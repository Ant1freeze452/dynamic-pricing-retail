# Dinamic Pricing Algorithm — MVP
---
 
## О проекте
 
Алгоритм динамического ценообразования для ритейла. Система автоматически рассчитывает оптимальную цену товара на основе исторических данных о продажах и цен конкурентов.
 
**Принцип работы:**
```
История продаж + Цены конкурентов
         ↓
  Модель спроса: Q(p) = A − B·p
         ↓
  Оптимальная цена: p* = (A + B·cost) / 2B
         ↓
  Прогноз: Старая цена → Новая цена → Прирост прибыли
```

## Быстрый старт
 
### 1. Клонировать репозиторий
```bash
git clone https://github.com/YOUR_USERNAME/dynamic-pricing.git
cd dynamic-pricing
```
 
### 2. Установить зависимости
```bash
pip install -r requirements.txt
```
 
### 3. Сгенерировать данные
```bash
python src/data_generator.py
# → data/sales_history.csv  (90 дней × 5 товаров)
```
 
### 4. Запустить алгоритм (в консоли)
```bash
python src/pricing_engine.py
# → data/pricing_report.csv
```
 
### 5. Запустить Streamlit-дашборд
```bash
streamlit run app.py
# → http://localhost:8501
```
 
### 6. Запустить тесты
```bash
pytest tests/ -v
```

## Что делает каждый модуль
 
### `src/data_generator.py`
Создаёт реалистичные синтетические данные:
- **Эластичность спроса**: `Q(p) = Q_base × (1 + e × ΔP/P)`
- **Эффект конкурента**: если его цена ниже → теряем часть покупателей
- **Сезонность**: выходные дни дают +20-35% продаж
- **Шум**: гауссовский, имитирует непредсказуемость рынка
### `src/pricing_engine.py`
Два уровня алгоритма:
 
| Уровень | Метод | Когда использовать |
|---------|-------|-------------------|
| Rule-based | Эвристические правила | Быстрые решения, нет данных |
| Оптимизация | Аналитический оптимум | Есть история, нужна точность |
 
**Математика оптимума:**
```
π(p) = (p − cost) × (A − B·p)    — прибыль
dπ/dp = 0  →  p* = (A + B·cost) / 2B
```
 
### `app.py`
Streamlit-дашборд с 4 вкладками:
1. **Данные и EDA** — обзор датасета, кривые эластичности
2. **Рекомендации** — таблица: Старая цена → Новая цена → Δ Прибыль
3. **Анализ товара** — детальный разбор выбранного товара
4. **Симуляция** — «перемотка времени»: применяем новые цены на N дней

## Ограничения модели
 
- Используется **линейная** модель спроса — на реальных данных спрос нелинеен
- Алгоритм не учитывает кросс-эластичность (как цена товара А влияет на продажи Б)
- Данные синтетические — реальные данные потребуют дополнительной очистки
- Модель предполагает стационарность — на практике паттерны меняются
